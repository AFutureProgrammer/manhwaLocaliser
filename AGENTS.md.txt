# Codex rules

Work small. Prefer minimal diffs over rewrites.

Before editing:
- Inspect only files needed for the task.
- Do not refactor unrelated code.
- Do not rewrite full files unless unavoidable.
- Preserve existing public API contracts.
- Preserve existing frontend/backend behavior unless the task says otherwise.

For bug fixes:
- Identify the exact code path first.
- Change the smallest viable surface.
- Do not add new features.
- Do not change memory, YOLO, or pipeline behavior unless directly required.

Validation:
- Run `python -m compileall backend memory` when backend changes.
- Run `npm run build` in `frontend/` when frontend changes.
- Final response: changed files + short summary + tests run. No full file dumps.
